# PACE v6.0 ヒアリング回答・確定事項

> 確定日: 2026-03-25
> ステータス: 確定版（全Q回答済み）

## 既存コードベースとの関係性

**方針: v6.0エンジンへの一本化（統合的進化）**

- `lib/engine/v6/` を新設し、オーケストレーターとする
- `conditioning/engine.ts` の ACWR → Node 4 の物理演算コンポーネントとして吸収（数式は v6.0 にアップデート）
- `assessment/` の CAT エンジン → Node 1 のデータ入力インターフェースとして存続
- 並行運用は禁止（判断の不一致を防ぐ）

## IMU / ウェアラブル連携

**方針: sRPE 先行のハイブリッド・インテーク**

- MVP: CSV アップロード + 手動入力
- Pro チーム向け: Catapult 等の外部 API コネクタを順次開発
- `device_kappa`: デバイス種別ごとの固定値マスタ（Catapult=0.9, GPS Watch=0.4, 手動入力=0.5）
- IMU なし時: デカップリング検知をバイパス、sRPE のみで推論

## 組織カテゴリ別半減期

**4層分類:**

| カテゴリ | 半減期 | 例 |
|---------|--------|-----|
| Metabolic (Muscle) | 3-7日 | 回復が早い |
| Structural-Soft (Tendon/Ligament) | 14-28日 | じわじわ蓄積 |
| Structural-Hard (Bone) | 60日〜 | 長期的リスク |
| Neuromotor (Proprioception) | 2-3日 | 短期的制御の乱れ |

## Node 0（EHR）MVPスコープ

- MVP: 手動入力（チェックボックス形式）のみ
- RiskMultiplier: CSV エビデンスに基づく臨床的デフォルト値（例: ACL既往=2.5x, 睡眠不足=1.8x）
- 管理画面から医療アドバイザーが調整可能

## マルチテナント

- 数理モデル: グローバル定数
- 判定閾値: チーム単位で ±20% オフセット可能
- RLS ロール: Player（自分のみ）/ Coach（チーム全員）/ Doctor（トレースログ含む全データ）

## 数理モデル導入の段階性

| フェーズ | モデル | 理由 |
|---------|--------|------|
| **MVP** | ODE 損傷修復 + EKF デカップリング | コア価値（破綻防止 + 嘘検知） |
| **Phase 2** | MRF 運動連鎖 + 応力集中テンソル | 文脈の接続（既往歴 + アライメント） |
| **Phase 3** | サンプルエントロピー | 高頻度 IMU 解析（計算負荷最大） |

## 計算実行環境

**方針: (C) Python サーバーレス（AWS Lambda / GCP Cloud Functions）**

- Edge Functions をゲートウェイ、計算ヘビーな処理のみ Python へ委譲
- SciPy / FilterPy / NetworkX / EntropyHub を使用
- 結果は JSON で受け取り

## パラメータ初期値

| 段階 | 方式 |
|------|------|
| Day 0 | CSV エビデンス + 先行研究（Banister モデル等）の固定値 |
| Day 1〜 | UI から手動微調整（感度設定） |
| Day 90〜 | ベイズ最適化による自動推定・更新 |
