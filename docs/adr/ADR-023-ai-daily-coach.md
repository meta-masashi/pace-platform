# ADR-023: AIデイリーコーチ — Gemini 2.0 Flash + ルールベースフォールバック

**ステータス:** 承認済み
**作成日:** 2026-03-24
**決定者:** @07-ml-engineer, @04-backend
**関連ADR:** ADR-009（LLMコンテキスト注入）, ADR-022（condition cache）

---

## コンテキスト

選手が毎日チェックインした後、「今日はどんなトレーニングをすべきか」というパーソナライズされたアドバイスを提供したい。

アドバイス生成方法の候補：

1. **完全ルールベース**: 閾値ベースの分岐でメッセージを生成
2. **LLM（毎回呼び出し）**: 常に最新の文脈でアドバイス生成
3. **LLM + ルールベースフォールバック**: LLM を優先しつつ、未チェックインや API エラー時はルールで補完

## 決定

**Gemini 2.0 Flash（`gemini-2.0-flash`）+ ルールベースフォールバックの組み合わせを採用する。**

### モデル選択理由

| モデル | 推論精度 | レイテンシ | コスト |
|---|---|---|---|
| Gemini 2.0 Flash | 十分 | < 2s | 低（Flash） |
| Gemini 2.0 Pro | 高精度 | 4〜8s | 高 |
| GPT-4o | 高精度 | 3〜6s | 高 |
| Claude 3.5 Sonnet | 高精度 | 3〜5s | 高 |

モバイルアプリのバックグラウンド取得で体験に大きく影響しないレイテンシと、月次 API コストのバランスから Gemini Flash を選択。

### ガードレール設計

```typescript
const SYSTEM_PROMPT = `
あなたはスポーツアスレティックトレーナーです。
- 医療診断・処方・受診勧奨は絶対に行わない
- NRS ≥ 7 の場合は必ずスタッフへの報告を促す
- 選手を傷つける可能性のある無謀なトレーニングを勧めない
- 返答は JSON 形式のみ（greeting/focus/tip/acwr_note）
`;
```

### フォールバック条件

以下の場合はルールベースカードを返す（Gemini API 節約）：
1. チェックイン未提出（その日のデータがない）
2. Gemini API エラー / タイムアウト（3回リトライ後）
3. JSON パース失敗

### ルールベースロジック概要

```typescript
function buildFallbackCard(input: CoachInput) {
  if (input.readiness < 40)  → "回復優先。今日は軽い散歩かストレッチのみ"
  if (input.acwr > 1.5)      → "過負荷注意。強度を 40% 以下に抑えて"
  if (input.nrs >= 7)        → "痛みが強い。スタッフに相談してください"
  default                    → "バランスの良いトレーニングを続けよう"
}
```

### コスト見積もり

- 1日あたり最大 1回/選手 の Gemini Flash 呼び出し
- 入力トークン: 約 400 tokens、出力: 約 150 tokens
- 100名規模チーム: 月約 $3〜5（Flash の price tier）

## 結果

- チェックイン後、ホーム画面に AIコーチカードが 1〜2秒で表示
- LLM エラー時もルールベースカードにより空白表示を防止
- `no_checkin` フラグで「チェックイン後にもっと詳しいアドバイスが届く」と促進

## トレードオフ

- Gemini API への依存: API 停止時はフォールバックで無停止
- 日本語品質: Gemini Flash は日本語品質が若干劣るケースがあるが許容範囲内
- プロンプトインジェクション: `SYSTEM_PROMPT` でロールを固定、選手入力はテキストではなく数値のみ注入
